# Лабораторная работа. Работа с GitHub  
Создание репозиториев, загрузка файлов, ссылки на прототипы Figma и совместная работа

##  Название проекта  
**3-decembrie — работы по Figma и управление репозиторием GitHub**

##  Краткое описание  
Репозиторий создан для выполнения лабораторной работы по GitHub.  
Содержит загруженные файлы, ссылки на макеты и прототипы Figma, структуру проекта, созданные ветки, Pull Request, Issues и примеры совместной работы через GitHub.

##  Цель работы  
Освоить создание и управление репозиторием в GitHub полностью в браузере:  
- загрузка материалов  
- работа с ветками  
- Pull Requests  
- Issues  
- добавление прототипов Figma  
- оформление README.md  
- организация совместной работы

##  Инструкции по использованию файлов  
- Все лабораторные работы размещаются в отдельных папках.  
- Файлы открываются напрямую из GitHub в браузере либо скачиваются.  
- Изображения и скриншоты доступны в разделе README или в папке images (если создана).  
- Figma-прототипы открываются по указанным ссылкам.

## Issues  
В проекте могут использоваться Issues для:  
- фиксации ошибок  
- мелких доработок  
- улучшения описания или документации  

Используемые метки:  
- enhancement  
- documentation  

## Pull Requests  
Работа с ветками организована через PR:  
1. Создана ветка: `feature-update-readme`  
2. Внесены изменения в README.md  
3. Открыт Pull Request  
4. Добавлено описание изменений  
5. PR отправлен на review и обновляется после каждого commit

## Совместная работа  
Добавление участников:  
- Settings → Collaborators → Add people  
- Назначение прав: Read / Write  
- Участники могут оставлять комментарии, открывать Issues и проверять PR.
